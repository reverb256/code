Today we're launching Twig, an AI code editor that connects to your product analytics. Write code that's informed by how real users actually behave.

## What is Twig?

Twig is an AI-powered code editor built on top of PostHog. It understands your users because it has access to your product analytics, session recordings, and feature flags.

Instead of just asking "write a function that does X," you can ask questions like:

- "Why are users dropping off at checkout?"
- "What's causing the spike in errors this week?"
- "Which users are affected by this bug?"

And Twig will analyze the data, find the answer, and help you fix it.

## How it works

Twig connects to your PostHog instance and indexes your codebase. When you ask a question, it:

1. Analyzes the relevant product data
2. Correlates it with your code
3. Suggests changes that address the actual problem

It's not magic—it's just bringing together data that was always there but never connected.

## Who is this for?

Twig is for developers who care about their users. If you've ever thought "I wish I knew what users were actually doing when they hit this code," Twig is for you.

It's particularly useful for:

- **Product engineers** who need to balance feature development with user experience
- **Full-stack developers** who own features end-to-end
- **Startup teams** where everyone wears multiple hats
